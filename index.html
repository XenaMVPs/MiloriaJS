MiloriaJS Framework

Introducción

MiloriaJS nace con el propósito de unificar reactividad, componentes, enrutamiento, SSR simulado y optimizaciones en un solo archivo: miloria.js. Con ello, se reduce la dependencia de configuraciones externas y se simplifica notablemente el flujo de trabajo.

Problemas que Soluciona

  • Complejidad creciente: Agrupa en un único sitio las funciones de reactividad, enrutamiento y SSR para evitar configuraciones dispersas.
  • Rendimiento y reactividad: Emplea un sistema avanzado (computed, watch, deepWatch) que minimiza renders innecesarios.
  • Enrutamiento isomórfico: Navegación con History API y SSR simulado en un único flujo continuo.
  • Partial Hydration: Hidrata solo lo que se requiere, acelerando la carga de páginas con contenido estático.
  • Sistema de plugins: Ofrece flexibilidad para extender la funcionalidad sin complicar tu código base.

Instalación y Uso

  1. Descarga el archivo miloria.js y colócalo en la carpeta de tu proyecto.
  2. Incluye la siguiente línea antes de cerrar </body> en tu HTML: 
    <script src="miloria.js"></script>
  3. Inicia tu aplicación definiendo componentes con Miloria.defineComponent y registrando rutas con Miloria.router.

Ejemplo de Uso

En el siguiente ejemplo definimos dos componentes para las rutas / y /about. El contenedor #app es donde se montará la aplicación principal: 


/* EJEMPLO (main.js por ejemplo) */

// Importamos funciones de Miloria
const { defineComponent, reactive, watch, onMount, router } = Miloria;

// Definimos un componente Home
const HomeComponent = defineComponent({
  name: "HomeComponent",
  setup() {
    const state = reactive({ contador: 0 });

    function incrementar() {
      state.contador++;
    }

    onMount(() => {
      console.log("HomeComponent montado");
    });

    watch(() => state.contador, (nuevo, viejo) => {
      console.log("El contador cambió de", viejo, "a", nuevo);
    });

    return () => `
      

        
Bienvenido a MiloriaJS

        
Contador: ${state.contador}

        
        
      

    `;
  }
});

// Definimos un componente About
const AboutComponent = defineComponent({
  name: "AboutComponent",
  setup() {
    return () => `
      

        
Acerca de MiloriaJS

        
Integra reactividad, SSR, partial hydration y rutas en un solo archivo.

        
      

    `;
  }
});

// Registramos rutas
router.register("/", HomeComponent);
router.register("/about", AboutComponent);

// Iniciamos el router con el contenedor #app
router.init("#app");

De esta forma, navegar a / o /about no recarga la página y cada ruta corresponde a un componente diferente.

API Principal Resumida

  • Miloria.defineComponent({ name, setup }) — Crea un nuevo componente.
  • Miloria.reactive(obj) — Convierte un objeto en reactivo.
  • Miloria.watch(getter, callback) — Observa cambios sobre un getter.
  • Miloria.router.register(path, component) — Registra rutas en el enrutador.
  • Miloria.ssr(config) — Renderizado en servidor (simulado).
  • Miloria.hydrateOnly(selector) — Hidrata solo la parte indicada del DOM.
  • Miloria.createStore(initialState) — Crea un store global reactivo.
  • Miloria.registerPlugin(plugin) — Permite extender el framework con plugins.
