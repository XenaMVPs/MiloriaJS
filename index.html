Introducción

MiloriaJS se presenta como una alternativa moderna y muy optimizada para construir aplicaciones web de alta complejidad con un enfoque minimalista. Inspirado en la filosofía de Vue, React y Svelte, MiloriaJS consolida en un único archivo ( miloria.js ) las funcionalidades esenciales para el desarrollo de frontends profesionales: Virtual DOM mejorado, reactividad profunda, ciclo de vida de componentes, enrutamiento isomórfico con lazy loading, SSR simulado, partial hydration, gestion de estado global y muchísimo más.

Nuestro objetivo es claro: ofrecer un flujo de trabajo simple y escalable en un solo archivo, para evitar la proliferación de herramientas externas.

Problemas que Soluciona

  • Configuraciones complicadas: Al integrar la reactividad, enrutamiento, SSR y animaciones en un solo archivo, MiloriaJS elimina la necesidad de configuraciones externas o múltiples dependencias.
  • Re-renderizados excesivos: Gracias al Virtual DOM mejorado y al sistema de trackeo profundo, se minimizan los re-renderizados. La reactividad está diseñada para ser eficiente incluso en apps de mediano y gran tamaño.
  • Arquitecturas dispersas: Al contar con un sistema de módulos internos, un EventBus global y un Store Avanzado con middlewares y persistencia, tu lógica se encuentra centralizada y bien organizada.
  • Falta de escalabilidad: Soporta mixins, extends, validación de props y plugins, asegurando que tu proyecto crezca sin volverse inmanejable.
  • Necesidad de SSR y optimizaciones: Con SSR simulado y partial hydration, MiloriaJS mejora la primera carga y el SEO, sin requerir un setup complejo.
  • Enrutamiento asíncrono e isomórfico: El lazy loading (o code splitting simulado) evita cargar todos los componentes al inicio, mejorando los tiempos de arranque de la aplicación.
  • Animaciones y transiciones limitadas: Con animate, animateAdvanced y transitionComponent, se pueden crear interacciones de alto nivel sin librerías externas.
  • Falta de herramientas Dev: MiloriaJS incluye logs avanzados, devTools y un modo debug para mejorar la experiencia de desarrollo, además de un sistema inicial de HMR (Hot Module Replacement) simulado.

Características Destacadas

  1. Virtual DOM más robusto: Soporte de key y un algoritmo de diff avanzado para reducir renders innecesarios.
  2. Reactividad mejorada: Trackeo profundo de dependencias, computed, watch, deepWatch, advancedComputed y error boundaries integrados.
  3. Sistema de Componentes enriquecido: Hooks completos (onBeforeMount, onMount, onBeforeUpdate, onUpdate, onBeforeUnmount, onUnmount), mixins, extends y validación de props.
  4. Enrutamiento Isomórfico Avanzado: Rutas dinámicas, lazy loading y middlewares para proteger rutas o ejecutar lógica previa.
  5. SSR Simulado y Partial Hydration: Ideal para mejorar SEO y acelerar la carga inicial sin configuraciones complejas.
  6. Store Global Avanzado: Soporta módulos, middlewares y persistencia local o remota (createStoreAdvanced).
  7. Animaciones y Transiciones: API de animaciones (animate, animateAdvanced) y transitionComponent para un UX enriquecido.
  8. Plugins y Hooks Globales: Extiende la funcionalidad de tu app con facilidad (registerPlugin, usePlugins, hooks beforeRender, afterRender, etc.).
  9. DevTools, Logging y Modo Debug: Logs detallados, warnings y errors, además de un modo debug para diagnosticar problemas sin complicaciones.
  10. HMR Simulado: Permite forzar remount de componentes y simular recargas en caliente para mejorar la experiencia de desarrollo.

Instalación y Configuración

MiloriaJS se integra a tu proyecto de manera muy sencilla. Solo sigue los pasos:

  1. Descarga el archivo miloria.js (o clona el repositorio).
  2. Inclúyelo en tu HTML: 
    <script src="miloria.js"></script>
  3. Configura la aplicación si lo requieres. Por ejemplo, puedes activar el modo debug: 
    Miloria.config.set({ debug: true });
  4. Inicia tu app definiendo componentes, rutas o creando tu store global según tus necesidades.

Eso es todo. MiloriaJS no necesita herramientas externas ni complicados sistemas de bundling.

Ejemplos de Uso

1. Definir Componente con Reactividad


const { defineComponent, reactive, onMount, watch } = Miloria;

const MyCounter = defineComponent({
  name: "MyCounter",
  setup() {
    const state = reactive({ contador: 0 });

    function incrementar() {
      state.contador++;
    }

    onMount(() => {
      console.log("MyCounter montado en el DOM");
    });

    watch(() => state.contador, (nuevo, viejo) => {
      console.log("Contador cambió de", viejo, "a", nuevo);
    });

    return () => Miloria.h("div", null,
      Miloria.h("h3", null, "Contador: " + state.contador),
      Miloria.h("button", { onclick: incrementar }, "Incrementar")
    );
  }
});

2. Montar un Componente


Miloria.mountComponent("MyCounter", document.getElementById("app"));

3. Uso de Enrutamiento Isomórfico con Lazy Loading


// Rutas principales
Miloria.router.register("/", "HomeComponent");
Miloria.router.register("/about", "AboutComponent", {
  loader: () => import("./about.js").then(m => m.default) // Ejemplo de lazy load
});

Miloria.router.setNotFound("NotFoundComponent");
Miloria.router.init("#app");

4. SSR Simulado


const mySSR = Miloria.ssr({ templateCache: true });
const htmlString = mySSR("MyCounter", { someProp: "valor" });
console.log("Resultado SSR:", htmlString);

5. Store Avanzado con Persistencia Local


const store = Miloria.createStoreAdvanced(
  { user: "Desconocido" },
  {
    persistKey: "miAppState",
    modules: {
      auth: {
        state: { token: null },
        mutations: {
          setToken(state, payload) {
            state.token = payload;
          }
        }
      }
    }
  }
);

// Uso de commit con modules
store.commit("auth/setToken", "ABC123XYZ");
console.log(store.state.auth.token); // "ABC123XYZ"

API Principal Resumida

  • Miloria.defineComponent({ ... }) — Crea un componente con setup, props, etc.
  • Miloria.mountComponent("Componente", contenedor) — Monta un componente en el DOM.
  • Miloria.reactive(obj) — Convierte un objeto en reactivo.
  • Miloria.watch(getter, callback) — Observa cambios de un getter.
  • Miloria.createStore(initialState) o Miloria.createStoreAdvanced(...) — Crea un store global.
  • Miloria.router.register(path, component, opts) — Registra rutas.
  • Miloria.ssr(config) — Inicializa renderizado en servidor simulado.
  • Miloria.hydrateOnly(selector) — Aplica partial hydration en un fragmento del DOM.
  • Miloria.registerPlugin(plugin) — Añade funcionalidades extras al framework.
  • Miloria.animate(element, props, duration), animateAdvanced(...), transitionComponent(...) — Animaciones y transiciones.
  • Miloria.validateData(schema, data) — Validación genérica de datos.
  • Miloria.enableHMR() — Activa HMR simulado para re-cargar componentes en caliente.
  • Miloria.config.set({ debug: true }) — Configura opciones globales como debug.

Para más detalles, revisa el archivo miloria.js completo, donde encontrarás comentarios y cada método documentado.
